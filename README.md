# Screen Diff Tool

画面キャプチャ（スクリーンショット）をフォルダ単位で比較し、差分率の一覧（CSV）と視覚的に確認できる HTML レポートを生成するデスクトップツールです。

## できること

- **旧フォルダ**と**新フォルダ**内の同名画像をペアで比較
- ピクセル単位の差分を検出し、差分率（%）を算出
- 差分領域を赤枠でハイライト（HTML では透明背景のオーバーレイとして重ね表示）
- 同名 `.txt` のテキスト比較（差異行数・行単位 diff）
- `report.csv` … 一覧表（Excel 等で開ける）
- `report.html` … ブラウザで新旧を並べて確認（オーバーレイ・点滅・拡大など）
- **AIプロンプト タブ** … 画像比較用プロンプトの編集（比較タブからクリップボードコピー可能）

## 必要環境

| 項目 | 要件 |
|------|------|
| Java | 17 以上 |
| ビルド | Apache Maven 3.x |

## ビルドと起動

```bash
# 依存関係の取得とコンパイル
mvn compile

# GUI を起動
mvn exec:java
```

JAR から起動する場合:

```bash
mvn package
java -jar target/screen-diff-1.0.0.jar
```

`mvn package` で依存ライブラリを含む実行可能 JAR（fat JAR）が `target/screen-diff-1.0.0.jar` に生成されます。

## 使い方

アプリは **比較**・**設定**・**AIプロンプト** の 3 タブで構成されています。

### 比較タブ

1. **旧フォルダ** … 比較の基準となる画像が入ったフォルダ
2. **新フォルダ** … 比較対象の画像が入ったフォルダ（旧フォルダと同じ相対パス・ファイル名に揃える）
3. **出力先** … レポートを書き出すフォルダ
4. **サブフォルダも比較する**（任意）… ON のとき、旧・新フォルダ以下のサブフォルダ内の画像も対象
5. **HTMLレポート作成** をクリック

比較処理のあと **`report.csv` と `report.html` が出力** されます。

- **画像比較用AIプロンプト** … AIプロンプト タブのプロンプトをコピー。`${OLD_DIR}` / `${NEW_DIR}` を比較タブの旧・新フォルダのフルパスに置換

各フォルダ横のフォルダアイコンから、指定パスをエクスプローラで開けます。

### 設定タブ

比較パラメータを変更します（次回起動時に復元）。

### AIプロンプト タブ

旧・新フォルダのパスを指定し、AI に画像ファイルを直接比較させるためのプロンプトを編集できます。初期テンプレートは `src/main/resources/ai-image-prompt.txt` に定義され、`${OLD_DIR}` / `${NEW_DIR}` / `${INCLUDE_SUBFOLDERS}` を記述します。比較タブの **画像比較用AIプロンプト** ボタンでコピーすると比較タブの旧・新フォルダ（フルパス）とサブフォルダ設定（ON/OFF）に置換されます。編集後の内容は `~/.screendiff_ai_image_prompt` に保存されます。

### 対象ファイル

次の拡張子の画像のみ処理します（大文字小文字は区別しません）。

- `.png`, `.jpg`, `.jpeg`, `.bmp`

| サブフォルダも比較する | 対象 |
|------------------------|------|
| OFF（既定） | 旧・新フォルダ**直下**の画像のみ |
| ON | 旧・新フォルダ以下を**再帰的**に走査 |

ペアは **旧フォルダからの相対パス** で一致させます（例: `screens/login.png`）。新フォルダに同じ相対パスが無い場合はスキップされ、ログに記録されます。同名 `.txt` も同じ相対パスで読み込みます。

### 比較パラメータ（設定タブ）

| 項目 | 範囲 | 既定値 | 説明 |
|------|------|--------|------|
| ブロックサイズ | 1〜20 | 1 | これより小さい矩形の差分は無視（ノイズ抑制） |
| しきい値 | 0〜255 | 10 | ピクセルごとの許容色差。大きいほど細かい差を無視 |
| 四隅の余白削除 | ON / OFF | OFF | 四隅の色に近い余白を除去してから比較 |
| 画像出力形式 | PNG変換 / JPEG変換 | PNG変換 | **HTML** に埋め込む旧・新画像の出力形式 |
| HTML画像配置 | HTML内に埋め込み / 別フォルダ | HTML内に埋め込み | HTML レポートの画像を 1 ファイルにまとめるか、`report_assets/` に分離するか |
| JPEG品質(%) | 10〜100 | 50 | JPEG変換時の圧縮品質（PNG変換のときは無効） |
| 画像を先頭から切り取る | ON / OFF | ON | 切取条件を満たす画像の先頭を切り取る |
| 切取条件(px) | — | 3000 | 画像高さがこの値より大きい場合に切り取りを適用 |
| 実際の切取量(px) | — | 1000 | 切り取り時に先頭から残す高さ |

### 出力ファイル

| ファイル | 内容 |
|----------|------|
| `report.csv` | ファイル名、ピクセル差異(%)、横幅の差異(px)、高さの差異(px)、切り取り、テキスト差異(行)、旧/新の幅・高さ |
| `report.html` | HTML比較レポート作成時 … 差分一覧サマリ（アコーディオン）と画像比較を含む |
| `report_assets/` | **HTML画像配置が「別フォルダ」のときのみ** … HTML 用の画像ファイル（旧・新・差分） |

**CSV のテキスト差異(行)** … 旧・新ともに同名 `.txt` がある場合のみ数値（差分行数＝削除・追加・変更を各1行で数えた件数。HTML テキスト差分表示と同じ）。無い場合は空欄。

## HTML レポートの機能

HTML比較レポート（`report.html`）の先頭に **差分一覧サマリ（アコーディオン）** があり、クリックで表示・非表示を切り替えられます。サマリのファイル名をクリックすると、該当の画像比較へスクロールします。

画面上部の固定ヘッダ（コントロール量に応じて高さ自動調整）から全画像に一括で適用できます。

- **差分一覧サマリ** … ヘッダーのリンクから差分一覧サマリへ戻る（折りたたまれている場合は自動で開く）
- **差分表示** … 差分矩形のみの透明オーバーレイを旧・新の上に重ね表示
- **新旧表示** … 反対側のキャプチャを半透明で重ねる
- **透明度** … オーバーレイの不透明度
- **交互表示** … 旧と新を一定間隔で切り替え（速度 10〜500ms、既定 250ms）
- **表示方法** … 両方 / 旧のみ / 新のみ（ラジオボタン）
- **拡大率** … 全画像の表示倍率（25〜200%、既定 100%）。クリック拡大とは独立。100% 超ではドラッグで表示位置を移動可能
- **縦長画像** … 「画像を先頭から切り取る」ON のとき、切取条件より大きい画像のみ先頭から実際の切取量まで使用（例: 3000px 超 → 先頭 1000px）
- **スライダ操作** … 拡大率・透明度・交互表示の速度は Ctrl+← / Ctrl+→ でレンジの 5% 刻み

**差分一覧サマリ（アコーディオン・ページ上部）**

- **並び替え** … サマリ表の列ヘッダをクリックで昇順・降順を切り替え（▲昇順 / ▼降順）。初回はファイル名が昇順、数値列は降順
- **切り取り** … 先頭切り取りが実際に適用された画像を表示（`—` / `旧` / `新` / `旧・新`）。該当画像の下にも「切り取り」と表示

**各比較項目**

- **テキスト差分（アコーディオン）** … 各比較項目の画像下に折りたたみ表示。旧・新 `.txt` がある場合は差分行のみ表形式で表示（一致行は省略。削除=赤系、追加=緑系、変更=旧赤/新緑）。変更行は行内の単語差分もハイライト表示（空白のみの差異は無視）
- **画像表示** … 通常はコンテナ幅に収める（実寸より小さい画像は拡大しない）。実寸以下の画像は中央寄せ
- **画像クリックで拡大** … クリック位置を中心に最大 **100%（実寸）** まで拡大（スライダーとは独立）。**拡大中にクリック** するとクリック拡大のみ解除（スライダーが 100% 超のときはその倍率に戻る）。拡大後は **ドラッグ** で表示位置を移動（**両方表示**のときは旧・新を同期）。Esc で拡大率スライダー・クリック拡大の両方をリセット

### ブラウザから PDF へ印刷

`report.html` をブラウザで開き、印刷（Ctrl+P）→「PDF に保存」で PDF 化できます。

印刷時は次の要素は**出力されません**（画面上では表示されたまま）。

- 固定ヘッダ（操作パネル）
- 差分一覧サマリ
- テキスト差分アコーディオン

各画像比較（`.diff-section`）は**1件ごとに改ページ**されます。差分オーバーレイ・拡大状態は印刷に含めず、旧・新の画像のみ出力します。

## 比較の仕組み

- [image-comparison](https://github.com/romankh3/image-comparison) ライブラリで 2 枚の画像を比較
- オプションで **四隅の余白削除** … 四隅付近の色を基準色として、近い色の行/列を余白とみなして削除（比較・HTML レポートにも反映）
- オプションで **先頭から切り取り** … 切取条件（px）より大きい画像のみ、実際の切取量（px）で上端を残して比較・レポート出力
- **テキスト比較** … 画像と同じベース名の `.txt`（例: `screen.png` → `screen.txt`）を旧・新フォルダから読み込み、[java-diff-utils](https://github.com/java-diff-utils/java-diff-utils) で行単位・行内（単語）diff を算出（空白の差異は無視）
- 解像度が異なる場合は、**大きい方の幅・高さ**に合わせ、不足分は白背景でパディング
- **ピクセル差異(%)** … 差分矩形が占める面積比率（しきい値・ブロックサイズ適用後）
- **横幅の差異(px)** … 旧・新の画像幅の差の絶対値
- **高さの差異(px)** … 切り取り前の旧・新の画像高さの差の絶対値（比較・HTML表示は切り取り後の画像を使用）
- 差分矩形は赤枠・半透明塗りつぶし（線幅 3px）。HTML の「差分表示」は新画像に合成せず、透明背景＋矩形のみのオーバーレイ

## 設定の保存

次のファイルはユーザーホーム直下に保存され、次回起動時に復元されます。

| ファイル | 内容 |
|----------|------|
| `~/.screendiff_history` | 旧 / 新 / 出力先フォルダの履歴（各最大 20 件）、サブフォルダ比較、ブロックサイズ、しきい値、四隅の余白削除、画像出力形式、HTML画像配置、JPEG 品質、先頭切り取り、切取条件、実際の切取量 |
| `~/.screendiff_ai_image_prompt` | AIプロンプト タブの画像比較用プロンプト |

## 注意事項

- **HTML画像配置「HTML内に埋め込み」（既定）** … 画像を Base64 で `report.html` に埋め込み、1 ファイルで完結します。枚数・解像度が多いとファイルサイズが大きくなります（1 比較あたり旧・新・差分の 3 枚）。JPEG 品質を下げると軽量化できます。
- **HTML画像配置「別フォルダ」** … 画像を `report_assets/` に書き出して HTML から参照します。HTML ファイルは小さく保てますが、配布時はフォルダごと必要です。
- 旧フォルダにだけ存在するファイルは比較されません。新フォルダのみのファイルも自動では拾いません。
- **大きい画像** … 比較時は最大約400万ピクセル・辺4096pxに自動縮小します（差分率は縮小後の画像基準）。HTML 画像配置が「別フォルダ」のときは `report_assets/` に書き出します。メモリ不足時は設定タブで **JPEG変換** を有効にするか、起動時に `java -Xmx4g -jar ...` などでヒープを増やしてください。
