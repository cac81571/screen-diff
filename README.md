# Screen Diff Tool

画面キャプチャ（スクリーンショット）をフォルダ単位で比較し、差分率の一覧（CSV）と視覚的に確認できる HTML / PDF レポートを生成するデスクトップツールです。

## できること

- **旧フォルダ**と**新フォルダ**内の同名画像をペアで比較
- ピクセル単位の差分を検出し、差分率（%）を算出
- 差分領域を赤枠でハイライト（HTML では透明背景のオーバーレイとして重ね表示）
- 同名 `.txt` のテキスト比較（差異率・行単位 diff）
- `report.csv` … 一覧表（Excel 等で開ける）
- `report.html` … ブラウザで新旧を並べて確認（オーバーレイ・点滅・拡大など）
- `report.pdf` … 印刷・共有向けの静的レポート（サイズ上限で自動分割）
- **AI タブ** … PDF 検証用プロンプトの編集・クリップボードコピー

## 必要環境

| 項目 | 要件 |
|------|------|
| Java | 17 以上 |
| ビルド | Apache Maven 3.x |

## ビルドと起動

```bash
# 依存関係の取得とコンパイル
mvn compile

# GUI を起動
mvn exec:java
```

JAR から起動する場合:

```bash
mvn package
java -jar target/screen-diff-1.0.0.jar
```

※ 実行時は依存ライブラリ（FlatLaf、OpenCSV、image-comparison、PDFBox）がクラスパスに必要です。単体 JAR にまとめる場合は `maven-shade-plugin` 等の追加設定が必要です。

## 使い方

アプリは **比較**・**設定**・**AI** の 3 タブで構成されています。

### 比較タブ

1. **旧フォルダ** … 比較の基準となる画像が入ったフォルダ
2. **新フォルダ** … 比較対象の画像が入ったフォルダ（ファイル名は旧フォルダと一致させる）
3. **出力先** … レポートを書き出すフォルダ
4. **HTML比較レポート作成** または **PDF比較レポート作成** をクリック

いずれのボタンでも比較処理のあと **`report.csv` が必ず出力** されます（HTML / PDF と同時）。

- **PDF検証用AIプロンプトコピー** … AI タブのプロンプトをクリップボードにコピー（PDF を AI に渡して検証するとき用）

各フォルダ横のフォルダアイコンから、指定パスをエクスプローラで開けます。

### 設定タブ

比較パラメータと PDF 出力設定を変更します（次回起動時に復元）。

### AI タブ

PDF レポートを AI にアップロードして旧新を突き合わせるためのプロンプトを編集できます。内容は `~/.screendiff_ai_prompt` に保存されます。

### 対象ファイル

旧フォルダ内の次の拡張子のファイルのみ処理します（大文字小文字は区別しません）。

- `.png`, `.jpg`, `.jpeg`, `.bmp`

新フォルダに同名ファイルがない場合はスキップされ、ログに記録されます。

### 比較パラメータ（設定タブ）

| 項目 | 範囲 | 既定値 | 説明 |
|------|------|--------|------|
| ブロックサイズ | 1〜20 | 1 | これより小さい矩形の差分は無視（ノイズ抑制） |
| しきい値 | 0〜255 | 10 | ピクセルごとの許容色差。大きいほど細かい差を無視 |
| 四隅の余白削除 | ON / OFF | OFF | 四隅の色に近い余白を除去してから比較 |
| 画像出力形式 | 変換なし / JPEG変換 | 変換なし | **HTML** に埋め込む旧・新画像の形式（PDF は元ファイルから出力） |
| JPEG品質(%) | 10〜100 | 50 | JPEG変換時の圧縮品質（変換なしのときは無効） |
| 画像2分割表示 | ON / OFF | ON | 縦長画像を先頭・末尾の2区間に分けて表示（HTML / PDF） |
| 先頭表示高さ(px) | — | 1500 | 2分割 ON 時、表示する先頭区間の高さ |
| 末尾表示高さ(px) | — | 1500 | 2分割 ON 時、表示する末尾区間の高さ |
| PDF上限サイズ(MB) | 0〜500 | 10 | 超過時 `report-01.pdf`, `report-02.pdf` … に分割（0=分割しない） |

### 出力ファイル

| ファイル | 内容 |
|----------|------|
| `report.csv` | ファイル名、ピクセル/画像サイズ/テキスト差異(%)、旧/新の幅・高さ |
| `report.html` | HTML比較レポート作成時 … 画像ペアと差分オーバーレイ。ブラウザで開く |
| `report.pdf` | PDF比較レポート作成時 … 上限内なら1ファイル。超過時は `report-01.pdf`, `report-02.pdf` … |

**CSV のテキスト差異(%)** … 旧・新ともに同名 `.txt` がある場合のみ数値。無い場合は空欄（HTML の差分フィルタでは対象外）。

## HTML レポートの機能

HTML比較レポート（`report.html`）では、画面上部の固定ヘッダ（コントロール量に応じて高さ自動調整）から全画像に一括で適用できます。

- **差分を重ねる** … 差分矩形のみの透明オーバーレイを旧・新の上に重ね表示
- **新旧を重ねる** … 反対側のキャプチャを半透明で重ねる
- **透明度** … オーバーレイの不透明度
- **交互表示** … 旧と新を一定間隔で切り替え（速度 10〜500ms、既定 100ms）
- **表示方法** … 両方 / 旧のみ / 新のみ（ラジオボタン）
- **表示ページ** … プルダウンで1画面だけ表示（「すべて」で全件。差分フィルタと併用可）
- **差分フィルタ** … ピクセル・画像サイズ・テキストそれぞれの差異率以上の項目だけ表示（`.txt` が無い場合はテキストフィルタ対象外）
- **縦長画像** … 画像2分割表示 ON かつ画像高さが先頭+末尾を超える場合、指定した先頭・末尾のみ表示し中間を「中略」バーで省略
- **スライダ操作** … 差分フィルタ 3 つは ← / → で 1% 刻み、Ctrl+← / Ctrl+→ で 5% 刻み。その他スライダは Ctrl+← / Ctrl+→ でレンジの 5% 刻み
- **テキスト差分（アコーディオン）** … 各比較項目の画像下に折りたたみ表示。旧・新 `.txt` がある場合は行単位の差分を表形式で表示（削除=赤系、追加=緑系、変更=旧赤/新緑）
- **画像表示** … 通常はコンテナ幅に収める（実寸より小さい画像は拡大しない）。実寸以下の画像は中央寄せ
- **画像クリックで拡大** … クリックで最大 **100%（実寸）** まで拡大（それ以上は拡大しない）。**拡大中にクリック** すると元に戻る。拡大後は **ドラッグ** で表示位置を移動（**両方表示**のときは旧・新を同期）。Esc でも元に戻す

## PDF レポート

PDF比較レポート（`report.pdf`）は印刷・共有向けの静的レポートです。

- 比較1件あたり2ページ（旧→新）。分割表示 ON かつ画像が分割対象の場合は4ページ（旧・先頭→旧・末尾→新・先頭→新・末尾）
- **ファイルサイズ上限** … 設定タブの「PDF上限サイズ(MB)」を超える場合、比較件数単位で `report-01.pdf`, `report-02.pdf` … に自動分割（上限内なら `report.pdf` の1ファイル）。1件だけでも上限超過の場合は警告付きで出力
- 各ページ上部 … 1行目 `旧　ファイル名` / `新　ファイル名`（分割時は `（先頭）` `（末尾）`）、2行目 `比較差異　ピクセル：…%　画像サイズ：…%　テキスト：…%`（HTML レポートと同形式）
- 画像2分割表示 ON の場合、PDF にも同じ先頭/末尾分割を適用
- 日本語表示には OS 標準フォント（Meiryo / MS Gothic 等）を使用

## 比較の仕組み

- [image-comparison](https://github.com/romankh3/image-comparison) ライブラリで 2 枚の画像を比較
- オプションで **四隅の余白削除** … 四隅付近の色を基準色として、近い色の行/列を余白とみなして削除（HTML / PDF レポートにも反映）
- **テキスト比較** … 画像と同じベース名の `.txt`（例: `screen.png` → `screen.txt`）を旧・新フォルダから読み込み、差異率を算出
- 解像度が異なる場合は、**大きい方の幅・高さ**に合わせ、不足分は白背景でパディング
- **ピクセル差異(%)** … 差分矩形が占める面積比率（しきい値・ブロックサイズ適用後）
- **画像サイズ差異(%)** … 旧・新の幅・高さの差から算出
- 差分矩形は赤枠・半透明塗りつぶし（線幅 3px）。HTML の「差分を重ねる」は新画像に合成せず、透明背景＋矩形のみのオーバーレイ

## 設定の保存

次のファイルはユーザーホーム直下に保存され、次回起動時に復元されます。

| ファイル | 内容 |
|----------|------|
| `~/.screendiff_history` | 旧 / 新 / 出力先フォルダの履歴（各最大 20 件）、ブロックサイズ、しきい値、四隅の余白削除、画像出力形式、JPEG 品質、画像2分割表示、先頭/末尾表示高さ、PDF上限サイズ(MB) |
| `~/.screendiff_ai_prompt` | AI タブの PDF 検証用プロンプト |

## プロジェクト構成

```
src/main/java/com/screendiff/
├── App.java                  # Swing GUI（メイン）
├── ImageComparator.java      # 画像比較ロジック
├── ImageMarginTrimmer.java   # 四隅余白削除
├── TextComparator.java       # .txt テキスト比較
├── ImageDisplaySplitter.java # 縦長画像の先頭/末尾分割
├── PdfReportGenerator.java   # PDF レポート生成
└── ReportGenerator.java      # CSV / HTML レポート生成
```

## 注意事項

- HTML は画像を Base64 で埋め込むため、枚数・解像度が多いとファイルサイズが大きくなります（1 比較あたり旧・新・差分の 3 枚）。JPEG 品質を下げると軽量化できます。
- 旧フォルダにだけ存在するファイルは比較されません。新フォルダのみのファイルも自動では拾いません。
- PDF の初回生成は日本語フォントの読み込みにより時間がかかることがあります。
